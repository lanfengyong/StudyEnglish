英语打卡 - iPad 随时使用（不依赖电脑）
====================================

部署后，iPad 只要有网络就能用，不需要家里电脑开着。

推荐两种免费方式，任选其一：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方式 A：Netlify 拖拽部署（最简单，约 5 分钟）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

适合：不想学 Git，最快上线

第 1 步：打包文件夹
  1. 右键 StudyEnglish 文件夹
  2. 发送到 → 压缩(zipped)文件夹
  3. 得到 StudyEnglish.zip

第 2 步：上传部署
  1. 电脑浏览器打开：https://app.netlify.com/drop
  2. 若没有账号，用邮箱免费注册
  3. 把 StudyEnglish.zip 拖进网页
  4. 等待上传完成，会得到一个网址，例如：
     https://random-name-123.netlify.app

第 3 步：iPad 安装
  1. iPad Safari 打开上面的网址
  2. 分享 → 添加到主屏幕 → 命名「英语打卡」
  3. 完成！以后点桌面图标即可

（可选）改一个好记的网址：
  Netlify 后台 → Site settings → Change site name
  例如改成 studyenglish2024 → https://studyenglish2024.netlify.app

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方式 B：GitHub Pages（稳定，适合长期用）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

适合：有 GitHub 账号，希望长期免费托管

第 1 步：注册 GitHub
  打开 https://github.com 注册账号

第 2 步：创建仓库并上传
  1. 点 New repository（新建仓库）
  2. 名字填：StudyEnglish
  3. 选 Public，创建
  4. 点 uploading an existing file
  5. 把 StudyEnglish 文件夹里所有文件拖进去
     （index.html、css、js、manifest.json、sw.js、icon.svg 等）
  6. 点 Commit changes

第 3 步：开启 Pages
  1. 仓库里点 Settings → Pages
  2. Source 选 Deploy from a branch
  3. Branch 选 main，文件夹选 / (root)
  4. 点 Save
  5. 等 1～3 分钟，页面会显示网址：
     https://你的用户名.github.io/StudyEnglish/

第 4 步：iPad 安装
  1. iPad Safari 打开上面的网址
  2. 分享 → 添加到主屏幕
  3. 完成

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
iPad 首次使用设置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 设置 → Safari → 麦克风 → 允许
2. 应用内「今日任务」→ 开启通知权限（可选）
3. 定期「学习记录」→ 导出 JSON 备份

音频文件说明
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

在线版音频也需要一起部署：
  - 把 MP3 放入 audio/ 各单元目录
  - 重新打包 zip 上传（Netlify）
  - 或重新上传到 GitHub（GitHub Pages）

没音频也能用，浏览器会自动朗读。

常见问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

问：换 iPad 后记录还在吗？
答：不在。记录存在浏览器里。请定期导出备份。

问：录音打分在 iPad 能用吗？
答：Safari 可能无法自动识别，支持录音回放后手动自评。

问：每日推送在 iPad 能用吗？
答：添加到主屏幕后效果更好；完全关闭后不一定准时推送。

问：网址会变吗？
答：Netlify/GitHub 免费网址一般不变；不要删除站点/仓库即可。
